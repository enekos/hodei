# Hodei v0.2 — Design Spec

## Overview

This iteration takes Hodei from a well-architected prototype to a usable keyboard-first tiling browser. It is structured in three phases: fix integration gaps, add navigation power features, then add workspaces and configuration.

**Scope**: Integration fixes, navigation features (URL bar, search-in-page, history, bookmarks), workspace management, TOML configuration.

**Non-goals**: Engine abstraction/swapping, Lua scripting, download manager, extensions system, multi-window support.

**Engine**: Servo. No abstraction layer — the `hodei-servo` crate is the engine interface.

---

## Phase 1: Integration Fixes

**Goal**: Every existing system works end-to-end. You can launch, browse, click links via hints, and save/restore sessions.

### 1.1 ViewId Coordination

**Problem**: `BspLayout` and `ViewManager` both generate ViewIds independently, risking desync.

**Fix**: `ViewManager` becomes the single source of ViewId generation. `BspLayout::split()` and related methods accept a `ViewId` parameter instead of generating one internally. Remove `next_split_id` from `BspLayout`.

**Files affected**: `layout.rs`, `view.rs`, `app.rs`

### 1.2 Session Save/Restore Wiring

**Problem**: `Action::SaveSession` and `Action::RestoreSession` in `app.rs` are TODO stubs, despite `SessionManager` being fully implemented.

**Fix**:
- **Save**: Collect all tile URLs/titles from `ViewManager`, serialize layout tree from `BspLayout`, pass to `SessionManager::save()`.
- **Restore**: Tear down all current tiles (close Servo views), call `SessionManager::restore()`, rebuild `BspLayout` from the returned layout tree, create new Servo views for each tile, navigate each to its saved URL.
- **Autosave**: On graceful quit (`:quit` command or window close), autosave to the "default" session.

**Files affected**: `app.rs`, potentially `session.rs` (if the restore return type needs adjustment)

### 1.3 Hint Click Completion

**Problem**: After a user selects a hint label, nothing happens. The JS evaluation result isn't routed back to complete the click.

**Fix**:
1. When entering Hint mode, inject `HINT_QUERY_SCRIPT` into the focused tile via `Engine::evaluate_js()`.
2. Store the JS callback result (list of `HintElement` with coordinates) in a new field on the app state.
3. As the user types in Hint mode, `InputRouter` filters labels. When a unique match is found, the app dispatches `Engine::click()` at the matched element's (x, y) coordinates.
4. Return to Normal mode after the click.

**Files affected**: `app.rs`, `input.rs` (Action variant may need the element data), `hint.rs` (already has `filter_labels`)

### 1.4 Page Metadata Propagation

**Problem**: `HodeiWebViewDelegate` receives title/URL callbacks from Servo but discards them.

**Fix**: Add a channel (`std::sync::mpsc` or a shared `Arc<Mutex<Vec<MetadataEvent>>>`) from the delegate to the app layer. On each frame tick, the app drains pending metadata events and:
- Updates the corresponding `View` in `ViewManager` (title, URL)
- If the updated view is the focused tile, refreshes the HUD

**Files affected**: `delegate.rs` (in hodei-servo), `app.rs`, `view.rs`

## Phase 2: Navigation Features

**Goal**: The keyboard interactions that make browsing fast — finding pages, searching content, managing history.

### 2.1 Enhanced Command Bar / URL Bar

Upgrade `:open` from a raw URL input into a smart navigation prompt.

**Behavior**:
- `:open <input>` triggers suggestion generation
- Suggestions sourced from: bookmarks (highest priority), history (fuzzy match on title + URL), then raw URL/search fallback
- If input is not a valid URL, treat it as a search query using the configured search engine (default: DuckDuckGo)
- Suggestions rendered as a list in the HUD below the command bar (up to 10 items)
- Arrow keys (or `Ctrl+n`/`Ctrl+p`) navigate suggestions, `Enter` selects, `Esc` cancels

**Fuzzy matching**: Simple substring + prefix scoring. No external fuzzy library needed — iterate candidates, score by: exact prefix > word boundary match > substring. Sort by score descending, break ties by recency.

**New types**: `Suggestion { url, title, source: Bookmark|History|Search }`, `SuggestionList` with selection index.

**HUD changes**: New suggestion list component in `hud.slint`. The command bar expands to show suggestions below it when in command mode with `:open`.

**Files affected**: `input.rs`, `hud.rs`, `hud.slint`, `app.rs`, new file `suggest.rs` in hodei-core

### 2.2 Search-in-Page

**New mode**: `Mode::Search` added to `InputRouter`.

**Keybinding**: `/` in Normal mode enters Search mode.

**Behavior**:
- User types a search query in the HUD search bar
- On each keystroke (or on `Enter`), inject JS to highlight matches in the page using `window.find()` API or a CSS highlight overlay approach
- `n` (in Normal mode after search) jumps to next match
- `N` jumps to previous match
- `Esc` clears highlights (inject JS to remove highlight styling) and returns to Normal mode
- Match count displayed in HUD: "3/17 matches"

**JS strategy**: Use `CSS.highlights` API if Servo supports it, otherwise fall back to wrapping matches in `<mark>` elements and scrolling to the active one.

**New actions**: `Action::SearchStart`, `Action::SearchNext`, `Action::SearchPrev`, `Action::SearchClear`

**Files affected**: `input.rs`, `types.rs`, `hud.rs`, `hud.slint`, `app.rs`

### 2.3 Browser History

**Storage**: New `history` table in SQLite:
```sql
CREATE TABLE history (
    id INTEGER PRIMARY KEY,
    url TEXT NOT NULL,
    title TEXT,
    visit_count INTEGER DEFAULT 1,
    last_visited_at TEXT NOT NULL
);
CREATE INDEX idx_history_url ON history(url);
CREATE INDEX idx_history_last_visited ON history(last_visited_at);
```

**Population**: Automatic. When Phase 1.4 metadata events fire a URL change (not just title change), insert or update the history entry (increment `visit_count`, update `last_visited_at`).

**Query**: `:history <query>` fuzzy-searches title and URL fields, displays results in the suggestion list. Selecting a result navigates to it.

**Pruning**: Cap at 10,000 entries. On insert, if over limit, delete the oldest entries by `last_visited_at`.

**New module**: `history.rs` in hodei-core

**Files affected**: `migrations/` (new migration), `session.rs` (shared DB connection), `app.rs`

### 2.4 Bookmarks

**Storage**: New `bookmarks` table in SQLite:
```sql
CREATE TABLE bookmarks (
    id INTEGER PRIMARY KEY,
    url TEXT NOT NULL UNIQUE,
    title TEXT,
    tags TEXT,  -- comma-separated
    created_at TEXT NOT NULL
);
CREATE INDEX idx_bookmarks_url ON bookmarks(url);
```

**Commands**:
- `:bookmark` — saves the current page (URL + title from focused tile)
- `:bookmark <tag1,tag2>` — saves with tags
- `:bookmarks <query>` — fuzzy search by title, URL, or tag
- `:bookmark-delete <url>` — removes a bookmark

**Normal mode**: `B` opens bookmark list (equivalent to `:bookmarks`).

**Integration with URL bar**: Bookmarks appear in `:open` suggestions with higher priority than history.

**New module**: `bookmarks.rs` in hodei-core

**Files affected**: `migrations/` (new migration), `input.rs`, `app.rs`

## Phase 3: Workspaces & Configuration

**Goal**: Persistent named workspaces and a TOML config file.

### 3.1 Named Workspaces

Evolve the session system into workspace management.

**Data model**: A workspace is a named session with additional metadata. The existing `sessions` table gains:
```sql
ALTER TABLE sessions ADD COLUMN is_active INTEGER DEFAULT 0;
ALTER TABLE sessions ADD COLUMN last_accessed_at TEXT;
```

**In-memory model**: `WorkspaceManager` wraps `SessionManager` and holds:
- `workspaces: HashMap<String, WorkspaceState>` — loaded workspaces (layout + tile URLs)
- `active: String` — name of the current workspace

**Switching**: When switching from workspace A to workspace B:
1. Save workspace A's current state to the HashMap (and mark dirty for next DB flush)
2. If B is in the HashMap, restore from memory (instant). Otherwise, load from DB.
3. Tear down current Servo views, rebuild layout, create new views, navigate.

**Commands**:
- `:workspace <name>` — switch to (or create) a workspace
- `:workspace-new <name>` — create an empty workspace and switch to it
- `:workspace-delete <name>` — delete a workspace (cannot delete the active one)
- `:workspace-list` — show all workspaces with tile counts in suggestion list
- `:workspace-rename <old> <new>` — rename a workspace

**Keybindings**: `Ctrl+1` through `Ctrl+9` switch to workspace by index (ordered by `last_accessed_at`).

**Autosave**: Active workspace is autosaved:
- On workspace switch
- On graceful quit
- Periodically (configurable interval, default 60s)

**Startup**: Restore the workspace where `is_active = 1`, or create a "default" workspace if none exists.

**New module**: `workspace.rs` in hodei-core (wraps `SessionManager`)

**Files affected**: `session.rs`, `app.rs`, `input.rs`, `migrations/`

### 3.2 TOML Configuration

**Location**: `~/.config/hodei/config.toml` (resolved via `dirs::config_dir()`).

**Schema**:
```toml
[general]
startup_url = "https://servo.org"
search_engine = "https://duckduckgo.com/?q={}"
autosave_interval_secs = 60
restore_workspace_on_startup = true

[keybindings]
# Override defaults. Format: action = "key"
# focus_left = "h"
# split_vertical = "ctrl+v"
# search = "/"

[appearance]
hud_opacity = 0.9
font_size = 14

[history]
max_entries = 10000
```

**Parsing**: New `config.rs` module. Uses `toml` crate (add dependency) and `serde::Deserialize`. A `Config` struct with `Default` impl provides all defaults. Missing file or missing fields fall back to defaults — the config file is entirely optional.

**Keybinding refactor**: `InputRouter` currently uses hardcoded match arms. Refactor to:
1. Build a `HashMap<(Mode, KeyCombo), Action>` from defaults
2. Override entries from TOML `[keybindings]` section
3. `InputRouter::handle_key()` does a HashMap lookup instead of matching

This makes keybindings data-driven and user-configurable.

**New dependency**: `toml = "0.8"`

**New module**: `config.rs` in hodei-core

**Files affected**: `input.rs` (keybinding refactor), `app.rs` (load config on startup), `Cargo.toml`

### 3.3 Startup Flow

Updated launch sequence:
1. Load TOML config from `~/.config/hodei/config.toml` (or defaults)
2. Initialize `InputRouter` with keybindings from config
3. Initialize `WorkspaceManager` with SQLite DB
4. If `restore_workspace_on_startup` is true, restore last active workspace
5. Otherwise, create "default" workspace with a single tile at `startup_url`
6. Start event loop

**Files affected**: `app.rs`, `main.rs`

## Database Connection Strategy

All modules that need SQLite access (`SessionManager`, `HistoryManager`, `BookmarkManager`, `WorkspaceManager`) receive a shared `Arc<Mutex<rusqlite::Connection>>` or, more simply, a single `Database` struct in a new `db.rs` module that owns the connection, runs migrations on startup, and exposes the connection for each manager's constructor. This avoids multiple connections to the same file and simplifies transaction handling.

## Migration Strategy

New SQL migrations run on startup via the `Database` initializer. Migrations are applied in order. The existing `001_init.sql` is unchanged.

## Implementation Ordering

Within each phase, features should be built in dependency order:
- **Phase 2**: History (2.3) and Bookmarks (2.4) first, then URL bar suggestions (2.1) which queries them, then Search-in-page (2.2) which is independent.
- The URL bar can be built incrementally: basic URL/search-fallback first, then suggestion sources added as history and bookmarks land.

## Testing Strategy

- Unit tests for all new modules (`suggest.rs`, `history.rs`, `bookmarks.rs`, `workspace.rs`, `config.rs`)
- Unit tests for refactored `InputRouter` with configurable keybindings
- Integration tests for session save/restore round-trip
- Integration tests for workspace switch cycle
- Existing 57+ tests must continue to pass (refactoring `layout.rs` ViewId generation will require test updates)
